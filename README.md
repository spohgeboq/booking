# 💇‍♀️ Онлайн-Система Записи (Booking System)

Комплексная платформа для салонов красоты, барбершопов и частных специалистов. Позволяет клиентам записываться на услуги онлайн, а администраторам и мастерам — удобно управлять графиком, финансами и базой клиентов через защищенную админ-панель.

Вся система **полностью докеризирована** и готова к деплою на любой VPS сервер одной командой.

---

## 🏗 Архитектура проекта

Проект состоит из трех изолированных сервисов и единой базы данных:

1. **`booking-site`** (Клиентская часть) — Приложение для клиентов (React + Vite). Каталог услуг, выбор мастера, времени и оформление записи.
2. **`admin-panel`** (Админ-панель) — Приложение для сотрудников (React + Vite). Ролевая модель доступа (`OWNER`, `MASTER`), аналитика, статистика доходов, управление персоналом и записями.
3. **`backend`** — Серверная часть (Node.js + Express + TypeScript). Реализует REST API, логику JWT-аутентификации, загрузку изображений (с оптимизацией через `sharp`) и отправку уведомлений в Telegram.
4. **`db`** — База данных PostgreSQL (используется ORM Prisma).

Между всеми контейнерами настроен **Nginx Reverse Proxy**, который безопасно маршрутизирует трафик и скрывает внутренние порты.

## 🚀 Деплой на VPS (Production)

Для запуска проекта на сервере вам потребуется только **Docker** и **Docker Compose**.

### 1. Клонирование и настройка
Склонируйте репозиторий на ваш сервер:
```bash
git clone https://github.com/spohgeboq/booking.git
cd booking
```

### 2. Настройка переменных окружения
Скопируйте пример файла конфигурации:
```bash
cp .env.example .env
```
Откройте файл `.env` любым текстовым редактором (например, `nano .env`) и **обязательно** измените пароли и секреты для безопасности:

- `POSTGRES_PASSWORD` — придумайте сложный пароль для базы данных.
- `JWT_SECRET` — сгенерируйте случайную строку для шифрования сессий.
- `TELEGRAM_BOT_TOKEN` и `TELEGRAM_GROUP_ID` — (опционально) для получения уведомлений о новых бронированиях.

### 3. Запуск контейнеров
Запустите сборку и старт всех сервисов в фоновом режиме:
```bash
docker compose up -d --build
```
> При первом запуске скрипты автоматически применят миграции базы данных Prisma.

### 4. Проверка статуса
Убедитесь, что все контейнеры имеют статус **Healthy**:
```bash
docker compose ps
```

Система будет доступна:
- **Клиентский сайт:** `http://ВАШ_IP_ИЛИ_ДОМЕН/`
- **Админ-панель:** `http://ВАШ_IP_ИЛИ_ДОМЕН:8080/`
- **API:** `http://ВАШ_IP_ИЛИ_ДОМЕН/api/`

## 🛠 Технологический стек

**Фронтенд:**
- React 18, TypeScript, Vite
- Tailwind CSS, Shadcn UI, Framer Motion (анимации)
- React Router DOM, Zustand (стейт-менеджмент)

**Бэкенд:**
- Node.js 22 (LTS), TypeScript, Express
- Prisma ORM
- PostgreSQL 15
- Sharp (обработка изображений)
- JWT (аутентификация)

**Инфраструктура:**
- Docker & Docker Compose
- Nginx (Reverse Proxy & Static Web Server)
- Alpine Linux (в качестве базовых легковесных образов)

## 💻 Локальная разработка (Без Docker)

Если вы хотите разрабатывать проект локально без запуска всех контейнеров, используйте встроенные инструменты:

1. **База данных:** Убедитесь, что у вас установлен локальный PostgreSQL. В корневом `.env` файле пропишите правильные `POSTGRES_USER` и `POSTGRES_PASSWORD`.
2. **Backend:**
   ```bash
   cd backend
   npm install
   npx prisma generate
   npx prisma db push
   npm run dev
   ```
3. **Frontend (Клиент):**
   ```bash
   cd booking-frontend
   npm install
   npm run dev
   ```
4. **Frontend (Админка):**
   ```bash
   cd frontend
   npm install
   npm run dev
   ```

## 🔒 Безопасность

- **Изоляция:** База данных доступна исключительно изнутри Docker-сети, порт 5432 закрыт снаружи.
- **Least Privilege:** Все Node.js сервисы и Nginx запускаются от имени непривилегированных пользователей (`appuser`, `nginx`), а не от пользователя `root`.
- **Nginx Security:** Включена защита от брутфорса (`limit_req`), скрыты серверные заголовки (`server_tokens off`).
- **Сборка:** Используется `Multi-stage build`, поэтому в финальные образы попадает только скомпилированный код без лишних исходников.
